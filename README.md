# Windows Setup Script

## Описание

Набор скриптов для первичной настройки Windows: `setup.bat` запускает установку с правами администратора, а `setup-script.ps1` настраивает Проводник и устанавливает приложения через `winget`.

Цели:

- убрать ручную установку базового софта;
- ускорить настройку нового ПК;
- сделать процесс установки воспроизводимым;
- пропускать приложения, которые уже установлены.

---

## Запуск

### Рекомендуемый способ

Запустить файл:

```text
setup.bat
```

Если скрипт запущен без прав администратора, появится UAC-запрос. После подтверждения откроется новое окно с правами администратора, и установка продолжится там.

### Запуск напрямую через PowerShell

```powershell
powershell -NoProfile -ExecutionPolicy Bypass -File .\setup-script.ps1
```

Такой запуск нужно делать из PowerShell, открытого от имени администратора.

---

## Состав

```text
setup.bat          # точка входа; запрашивает права администратора и запускает PowerShell-скрипт
setup-script.ps1   # основной PowerShell-скрипт настройки и установки приложений
README.md          # описание проекта и инструкция по запуску
```

---

## Что делает скрипт

Перед установкой `setup-script.ps1`:

1. проверяет запуск от имени администратора;
2. проверяет наличие `winget`;
3. включает отображение скрытых файлов и папок в Проводнике;
4. включает отображение расширений файлов;
5. для каждого приложения проверяет, установлено ли оно уже;
6. если приложение найдено, пропускает установку;
7. если приложение не найдено, устанавливает его через `winget install`.

> Важно: текущая версия скрипта перезапускает `explorer.exe`, чтобы настройки Проводника применились сразу. На некоторых системах рабочий стол/Проводник может появляться с задержкой. Если это мешает, лучше убрать перезапуск Проводника из `setup-script.ps1` и просто открыть новое окно Проводника после выполнения скрипта.

## Требования

- Windows 10 или Windows 11;
- установленный `winget` / App Installer;
- доступ к интернету;
- права администратора.

`setup.bat` сам запрашивает права администратора через UAC. Если запускать `setup-script.ps1` напрямую, PowerShell-скрипт только проверит права и завершится с ошибкой, если они не выданы.

---

## Возможные проблемы

### `winget` не найден

Установи **App Installer** из Microsoft Store и перезапусти PowerShell/терминал.

### В `.bat` отображаются битые русские символы

Проверь кодировку файла. Для `.bat` безопаснее использовать английский текст или сохранить файл как `UTF-8 без BOM`.

### После настройки Проводника рабочий стол долго не появляется

Это связано с перезапуском `explorer.exe`. Более стабильный вариант — не перезапускать Проводник из скрипта, а применить настройки и открыть новое окно Проводника вручную.
